Kesimpulan Uji Coba

Berdasarkan pengujian yang dilakukan dengan mengisi data dan menekan tombol "Kirim" serta "Reset" pada masing-masing form, diperoleh kesimpulan sebagai berikut:

  • Tombol Reset: Mengembalikan nilai input ke nilai awal yang ditentukan pada atribut value (yaitu "Sedi" dan "Pamir").
  • Tombol Kirim: Mengirimkan data ke URL yang ditentukan pada atribut action (dalam kasus ini action="" berarti halaman itu sendiri).
  • Perbedaan Method GET dan POST:
    • GET: Data dikirim melalui URL (query string). Setelah submit, URL berubah menjadi ?nama_depan=...&nama_belakang=.... Data terlihat jelas di address bar.
    • POST: Data dikirim dalam body HTTP request, tidak terlihat di URL. URL tetap bersih (tidak ada query string). Cocok untuk data sensitif.
  • Perbedaan Target:
    • target="_self": Hasil submit ditampilkan di tab/jendela yang sama (halaman saat ini).
    • target="_blank": Hasil submit dibuka di tab baru (atau jendela baru, tergantung pengaturan browser).
  • Kombinasi: Dengan GET + _blank, tab baru terbuka dan URL berisi data. Dengan POST + _blank, tab baru terbuka namun URL tidak menampilkan data.
Secara umum, atribut method mengontrol cara pengiriman data, sedangkan target mengontrol tempat tampilan respons.
